Signals

  • Reactive : Angular track changes
  • UI-Update : Auto update UI when value changes
  • Chnage Dection : Fine Grained(only affected parts)
  • Prformance : Faster, Optimized
  • Syntax : count()
  • Update : set() / update()
  • Types : Writable Signal, Computable Signal

Key Constructs of the Signals API

The core Angular Signals API includes three primary functions:

  • Signal() : Creates a writable signal for managing local, synchronous state. You can modify its value using the .set() or .update() methods.
  • Computed() : Creates a read-only signal whose value is derived from the values of other signals. It automatically recalculates its value when any of its dependent signals change and is both lazily evaluated and memoized for performance.
  • Effect() : Used to run side effects (e.g., logging, updating external APIs like localStorage, or manual DOM manipulation) whenever one or more signals it depends on change. Effects run automatically when their dependencies are updated.
